Component kit

A single shared presentational kit that collapses the 20 bespoke per-template ui.tsx copies and the declarative engine's cards into canonical, themeable bricks.

What it is / why

Rooms had two rendering worlds that drifted apart. The declarative schema engine renders catalog rooms from a schema through one shared RecordCard — consistent, but generic. The 20 builtin templates each hand-rolled their own cards in a private templates/<name>/components/ui.tsx: 20 copies of Avatar, MoneyCents, and the same card layout, each drifting independently. The named symptom was a Trip Split expense card that rendered who paid, how much, who owes as one continuous run of muted 13px text — all the facts present, no visual hierarchy, unanswerable in under two seconds.

The fix in one line One shared presentational kit at @/components/kit that both the engine and the bespoke templates render with — fix a brick once, every room improves.

The kit is presentational only: each brick takes plain data props, holds no room logic or store access, and renders through shared CSS tokens. It was built incrementally by the brick-polish loop — each iteration takes one duplicated pattern, perfects it against a rubric (scannable in <2s, token-pure, overflow-safe, accessible, solo-first/proxy-aware), migrates every consumer, and deletes the per-template copy. Templates import from the barrel and never re-implement a brick.

The bricks

Nine bricks ship today, exported from kit/index.ts. Props below are the typed contract in summary — see each source file for the full signature.

BrickProps (summary)Replaces / used by
Avatar person?:{name,color} or scalar name/color; large, title Initials circle over the person's accent, with a ? fallback for unnamed proxy rows. Replaced 20 hand-rolled copies; used by every room.
MoneyAmount amount (major units) or cents; currency, className Sign-colored, tabular-figure money render. Replaced per-template Money/MoneyCents; used by the money rooms (Trip Split, RoomLedger, GroupFund, CoParent, ChoreBoard).
PersonChip person?:{name,color} or name; fallback (raw id), className Inline color dot + name so a raw id never leaks; neutral dot on fallback. Used by the engine's RecordCard (person field, status-by, created-by) and inline attribution.
RecordRow leading, title, meta?, trailing? slots; onClick?, ariaLabel?, className, style The canonical "actor did X · amount" row card; becomes an accessible <button> with centralized keyboard/focus handling when onClick is set. Used by every record list across the 4 money rooms.
SplitView members[] (each person/name/fallback + optional amountCents), currency? Replaces the split: a, b, c muted comma run with wrapping PersonChips, each showing that person's share. Drops into RecordRow.meta; used by Trip Split and RoomLedger.
MetaLine items?[] (joined by ·, falsy dropped) or children; as, className The muted 13px secondary line, replacing inline style={{fontSize:13}} across 21 templates. Structured segments compose conditionally without manual separators.
EmptyState icon?, title?, children (message), action?, className The teaching empty state with a solo-first "here's the one thing to do first" CTA slot. Replaces the bare <div class="empty"> blocks every room hand-rolled.
StatCard label?, value, sub?, tone?:"pos"|"neg", children? A headline summary stat (trip total, household total, fund total) replacing the inline big-number pattern; tone color-codes directional figures.
SectionHeader title, count?, action?, className A section title with an optional count and a trailing action button, replacing the hand-rolled .spread-wrapping header rows. Bare titles stay a plain section title.

Design tokens

Every brick renders through globals.css tokens only — no hardcoded hex, no magic spacing — so a room takes its own tint for free: each room's accent sets --template-accent, and the bricks (focus rings, dots, emphasis) follow it. The key classes per brick:

BrickClasses
Avatar.avatar (.lg for large)
MoneyAmount.money (tabular figures) + .amount-pos / .amount-neg
PersonChip.person-chip, .person-chip__dot, .person-chip__name
RecordRow.card .row + .record-row (--clickable for the button form)
SplitView.split-view, .split-view__item, .split-view__amount
MetaLine.meta-line, .meta-line__sep
EmptyState.empty, .empty__icon, .empty__title, .empty__action
StatCard.stat-card__value (+ .amount-pos/neg for tone)
SectionHeader.section-header, .section-title, .section-header__count
