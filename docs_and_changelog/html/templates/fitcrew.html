Fit Crew

A friend workout league in a private room — log sessions, chase streaks, top the weekly board, and bet silly prizes.

The problem

Staying consistent with workouts is easier with friends, but the existing options are heavy: Strava-style apps are global social networks with accounts, feeds, and your activity data on someone else's servers. A small crew of friends just wants to see who showed up this week, keep streaks honest, and put a brunch bet on the line.

The solution

Fit Crew is a Rooms template — Strava-ish for a small crew, not a global network. One person creates a crew room and shares an invite code. There are no accounts; each friend joins on their phone and picks their member profile. Everything (logs, streaks, prizes) lives in a Yjs CRDT document on each device and syncs end-to-end encrypted through the dumb relay, which can never read it.

One-line pitch Log workouts with friends, watch the weekly leaderboard and live streaks, and award prizes — no accounts, no server that can read your data.

Key flows

  1. Create — pick the Fit Crew template, name the crew, add at least two members.
  2. Share — send the room code (or use room invites); each friend joins and picks their own profile on their phone.
  3. Log a workout — choose an activity (run, gym, walk, yoga, sport, other), optionally add minutes, a note, and a proof photo.
  4. Check the board — the weekly leaderboard ranks members by workouts since Monday, with a live streak counter per member and a crown for #1.
  5. Prizes & stakes — anyone can add a stake (title, emoji, description); anyone can award it to a member from a dropdown once it's earned.

The main screen also shows a personal hero strip (current streak / workouts this week / best-ever streak) and a recent-activity feed of the last 12 logs with proof photos inline.

Data model (template.fitcrew.*)

All state lives in the public Yjs doc under the fitcrew template branch — there is no separate admin data beyond the standard room shell.

KeyShapeHolds
template.fitcrew.crewMap{ name, createdAt } — crew settings; a set name marks the room as initialized.
template.fitcrew.membersMap<id, Member>{ id, name, color, joinedAt } — sorted by join time.
template.fitcrew.logsMap<id, WorkoutLog>{ memberId, activity, minutes?, note?, proofImage?, dayKey, loggedAt }.
template.fitcrew.prizesMap<id, Prize>{ title, emoji, description?, createdById, createdAt, awardedToId? }.

Two fields do extra work: dayKey is the local calendar day (YYYY-MM-DD) stamped at log time and is the only input to streak math, and proofImage is a serialized image-brick JSON value (the photo is stored inside the doc, then the room is compacted after inline upload).

Streaks and the weekly board

Nothing is precomputed or stored — stats are derived client-side from the logs on every render (lib/streaks.ts):

  • Current streak — count of consecutive distinct dayKeys ending at today or yesterday. Anchoring on yesterday means you don't see your streak drop to zero before you've had a chance to work out today.
  • Best streak — the longest consecutive-day run anywhere in history (never less than the current streak).
  • Weekly count — raw number of logs since Monday 00:00 local time (Sunday counts as the end of the week, not the start).
Leaderboard tie-breaking The board sorts by workouts this week, then current streak, then best streak. The top row gets the crown.

Notable rules and edge cases

  • Two logs in one day count once for streaks — streaks use distinct day keys, but each log still counts toward the weekly total.
  • Local time everywhere — dayKey and the Monday week boundary use the device's local clock, so crews across time zones may see slightly different "weeks". Acceptable for a friend group.
  • No edit/delete on logs — v1 is append-only; honesty is socially enforced (that's what proof photos are for).
  • Prizes are open — anyone can create a prize and anyone can award (or re-award) it via the dropdown; there is no owner gate. The awarded member shows a trophy pill.
  • Setup is owner-only — until the owner finishes setup, joiners see a "connecting" screen; after setup, joiners without a profile go through the profile picker.
  • Invite slots become members — reserving invites pre-creates member entries (slotId becomes the member id), so the invited friend lands directly on their profile.
