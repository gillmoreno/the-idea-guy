Trip Split

A Splitwise-style room for trips: friends log shared expenses, the app computes who's up and who's down, and suggests the minimum set of "X owes Y" transfers to settle.

The problem

On a group trip, one person pays for dinner, another covers the taxi, a third books the rental — and by day three nobody knows who owes whom. Mental math and group-chat IOUs don't scale, and the popular fix (Splitwise) means handing a third-party server a complete record of your group's spending.

The solution

Trip Split is the second Rooms template. A trip is a room: travelers join with the invite code, anyone logs expenses, and every device independently computes net balances and simplified settlement suggestions from the same synced expense list. Local-first like all Rooms — Yjs CRDT on each phone, end-to-end encrypted relay sync, no server-side expense database. Actual settling happens outside the app (Venmo, cash); Trip Split just tells you the minimum transfers.

Key flows

  • Create & join — pick Trip Split, name the trip, add travelers (minimum 2). Share the room code; friends join from the Rooms home screen, and each device picks "who's on this device" via the profile picker. The creator holds the room's admin secret (shell-level role), but all v1 trip data is on the public channel — any traveler can add or delete expenses.
  • Log an expense — description, amount, who paid, and which travelers it's split among (equal split). Date defaults to today; expenses list newest-first by date then creation time.
  • Check balances — the Balances tab shows each traveler's net position (positive = others owe them) plus the simplified debt list, e.g. "Ana owes Ben $42.50".

The UI is two tabs (TripView): Expenses and Balances, on the shared AppTabBar.

Balance math

Two pure functions in lib/balances.ts do all the work, recomputed from scratch on every render — there's no stored balance state to drift out of sync:

1. computeBalances For each expense, the amount is split equally among splitAmongIds. Each non-payer's share decreases their net and increases the payer's net by the same amount. (If the payer is in the split, their own share simply cancels out — it's skipped.)
2. simplifyDebts Greedy matching, Splitwise-style: split members into creditors (net > 0) and debtors (net < 0), sort both largest-first, then repeatedly pay the smaller of the two heads until everyone is at zero. Produces a short, human-readable list of transfers rather than a web of pairwise IOUs.

Currency & rounding

  • All amounts are stored as integer cents (amountCents) to avoid floating-point drift.
  • One currency per trip, chosen at setup (default USD); it's a display label, not converted.
  • Uneven divisions are handled deterministically: each splitter gets floor(amount / n), and the leftover cents are distributed one each to the first splitters in the list — so a $10.00 three-way split charges 334 / 333 / 333 cents and always sums exactly to the total.

Data model (Yjs keys)

Public branch only in v1 — there is no admin-only trip data.

KeyHolds
template.tripsplit.trip{ name, currency, createdAt }
template.tripsplit.travelersMap of { id, name, color, joinedAt }
template.tripsplit.expensesMap of { id, description, amountCents, paidById, splitAmongIds[], date, createdAt, createdById }

Notable rules & edge cases

  • Balances ignore split ids that don't match a known traveler (defensive filter in computeBalances), so a removed or unknown id can't corrupt totals.
  • Expenses record createdById separately from paidById — anyone can log an expense on someone else's behalf.
  • Deleting an expense is the only mutation besides adding; balances simply recompute.
  • Intentional v1 limits: equal splits only (no shares/percentages), single currency, no recorded settlements (paying someone back doesn't zero the ledger — you'd delete or offset expenses), no categories/receipts, and travelers are added at setup only. Unequal splits, settlements, and mid-trip traveler add are the planned next steps.

Key code paths

  • apps/rooms/web/src/templates/tripsplit/lib/balances.ts — computeBalances + simplifyDebts (all the money logic).
  • apps/rooms/web/src/templates/tripsplit/lib/store.ts — TripSplitStore: Yjs CRUD for trip, travelers, expenses.
  • apps/rooms/web/src/templates/tripsplit/lib/types.ts — data shapes (integer-cents convention documented here).
