User flows

How a family gets from zero to kids earning allowance.

1. Create family (parent)

  1. Welcome → Create family → generates family code + parent secret (stored on device).
  2. Setup: name, currency, payday, members, seed chores.
  3. Publish catalog + permissions to public channel.

2. Kid joins

  1. Join with family code only (QR or paste).
  2. Wait for public data if parent still setting up.
  3. Pick profile → optional device link paste → kid dashboard.

3. Daily use

  • Kid — see chores (and rewards if allowed), tap done, balance updates after approval.
  • Parent — Approvals tab: completions + chore proposals; Chores tab: catalog; Payday: mark paid.

4. Parent on second device

Join with family code → pick parent profile → enter parent secret (ParentGate) → admin sync starts.

5. Approvals logic

  • Chores with requiresApproval → completion stays pending until parent approves.
  • Otherwise auto-approved.
  • Penalties: parent-only, immediately approved negative amount.
