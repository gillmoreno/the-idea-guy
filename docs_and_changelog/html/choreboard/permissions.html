Kid permissions

Parents configure what each kid sees and can do — synced to kid devices on the public channel.

Not the same as crypto access

  • Parent secret — stops kids from changing admin data.
  • Permissions — control what the official app shows (rewards, balances, buttons).

Defaults (all kids)

SettingDefault
See reward on each choreOn — kids see e.g. $0.50
See own balance / pending / weekOn
Mark done / suggest chores / historyOn
See sibling balancesOff

Per-kid overrides

Settings → Kid permissions → pick a child. Changes publish to public permDefaults / permOverrides.

Honest limit

A modified client with the family code could ignore permissions and read public sync data. For normal use, UI policy is enough; use device links + signatures for stronger accountability on approvals.
