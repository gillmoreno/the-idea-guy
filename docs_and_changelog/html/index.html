Plain-language explanation of why we built things this way — not a line-by-line code reference. Kept in sync with the repo via a manifest and git hooks.

What lives here

  • ChoreBoard — family chores + allowance, local-first, encrypted sync.
  • Local-First Kit — reusable sync/encryption pattern for future mini-apps (documented inside ChoreBoard pages).

ChoreBoard at a glance

Problem Parents need chores tied to allowance; kids need motivation; data should stay in the family.
Approach Installable web app (PWA). Data on devices. A small relay only moves encrypted blobs — never reads chores or amounts.
Roles Parents hold an extra parent secret for admin actions. Kids use a family code only. Optional per-kid permissions and device links.
