How these docs work

Manifest + git hooks + Claude Code skills keep this HTML site aligned with the code.

Files

After code changes (sync)

  1. You commit code as usual.
  2. The post-commit hook runs scripts/check-html-docs-sync.sh.
  3. If sources changed but their HTML page did not, it writes .cursor/docs-sync-reminder.json.
  4. In Claude Code say: "sync the HTML docs" — the html-docs skill reads the reminder and updates only the stale pages.

New feature (bootstrap)

Add an entry to doc-manifest.json (or just describe the feature) and say "bootstrap HTML docs for <feature>". The skill reads the sources and writes a new page, adds it to every page's nav, and registers it in the manifest.

Install git hooks (once per clone)

./scripts/install-git-hooks.sh

Design rules for writers (human or agent)

  • Explain decisions and user-visible behavior, not every function name.
  • One HTML page per feature area; link between pages; every page carries the shared nav.
  • End every article with a .footer line: sources + "Last synced" date.
  • Deep specs stay in docs_and_changelog/*.md — these pages point to them, not duplicate them.
